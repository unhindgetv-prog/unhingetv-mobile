## App Privacy — Nutrition Label answers

Configure under **App Information → App Privacy → Edit** in App Store Connect. UnhingeTV does **NOT** use any tracking SDKs (verified: no PostHog, no Google Analytics, no Meta SDK, no advertising SDKs in either `web/package.json` or `mobile/package.json`). Mark **nothing** under "Used to Track You."

### Data Collected — Linked to User

| Category | Data Type | Purposes | Used for Tracking? | Rationale |
|----------|-----------|----------|--------------------|-----------|
| Contact Info | Email Address | App Functionality, Account Management, Customer Support | **No** | Required to create the user account, send transactional email (receipts, password resets), and respond to support tickets. Never sold, never shared with third parties, never used for cross-app/site tracking. |
| Identifiers | User ID | App Functionality | **No** | Server-side cuid that ties the user to their watch history, subscription status, and watchlist. Not shared with third parties. |

### Data Collected — Not Linked to User

| Category | Data Type | Purposes | Used for Tracking? | Rationale |
|----------|-----------|----------|--------------------|-----------|
| Usage Data | Product Interaction | App Functionality, Analytics | **No** | Sentry captures aggregated session/breadcrumb data for diagnostics. Not joined to user identity in Sentry; user IDs are explicitly scrubbed via `beforeSend`. |
| Diagnostics | Crash Data, Performance Data | App Functionality, Analytics | **No** | Sentry crash and performance telemetry. Not linked to user identity, not used for advertising. |

### Data NOT Collected
- Health & Fitness, Financial Info (Apple handles IAP — we never see card data), Location, Sensitive Info, Contacts, User Content (beyond comments/ratings which fall under "User Content" — see note), Browsing History, Search History, Audio Data, Photos/Videos, Other Data.

**Note on Comments/Ratings:** these are first-party user-generated content stored in our DB and are linked to the user's account. If Apple requires disclosing them, declare under **User Content → Customer Support / Other User Content**, purposes: **App Functionality**, tracking: **No**. (Current ASC questionnaire typically does not require this — disclose only if prompted.)

### Apple IAP
Purchase data flows directly from StoreKit to Apple. Apple's automated reporting covers this — **no manual declaration needed under Purchases**.
